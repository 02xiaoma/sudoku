这是我给杨咩咩写的解数独问题的小程序。
解决数独问题网上有比较多的例子，整体分为四步：
	1. 在图片中找出数独的区域；
	2. 将数独中的数字识别出来；(到此时已经生成棋盘)
	3. 解数独问题；
	4. 画出最后的解。
其中各个步骤的实现方法：
	1. 网上别人的方法都是通过识别最大的轮廓、最大的矩形、直线检测等识别出数独的区域。但是杨咩咩给我发的截图，其边缘并不明显，我数字图像处理没学过，所以没能检测出来。由于只针对杨咩咩这一个问题，所以把坐标找出来然后把相应的区域的坐标写死。
	2. 这一步按理说应该是通过机器学习的方法识别数字的。很多这样的例子。但是我这里没有很多样本来让我训练。我用matlab自带的OCR库发现效果很差。最后，我发现，游戏里面的数字都是一样的，而且在游戏画面的左边有9个数字的图片，所以可以暴力得用corr2来与这9张图片比较来解决。这里注意在用corr2之前要把数字周围的空白来删除，去一下噪声，然后把图片都调整为32*32大小之后再调用corr2。
	3. 我之前做leetcode的时候，第37题是解数独的。将这个程序封装一下，加一点命令行的参数。即可
	4. matlab有一个用整数规划解数独的例子，里面有画数独解的函数，可以直接拿来用。
下一步待改进的地方：
	1. 识别轮廓改进为用机器学习的方法来做，以使得可以更智能、更通用。
	2. 识别数字改为机器学习的方法。
	3. 考虑其他的解数独的方法来改进性能。我现在是直接用backtrack实现，感觉效率太差。
	4. 考虑其他的表现形式，比如把原始的数字和解出来的数字颜色画得不一样。
	5. 最后，考虑将整个程序封装，做成可以发布的形式
